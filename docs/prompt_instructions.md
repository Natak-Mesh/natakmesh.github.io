# Prompt Instructions for Natak Mesh Website Development

## Core Principles

1. **Keep Code Simple and Minimal**
   - Implement only what is explicitly requested
   - Avoid adding unasked-for features or "nice-to-haves"
   - Prioritize readability over clever solutions

2. **Documentation and Clarity**
   - Include clear, concise comments for all non-obvious code
   - Document the purpose of functions, classes, and complex logic
   - Use descriptive variable and function names

3. **Clean Code Practices**
   - Follow consistent formatting and style
   - Keep functions small and focused on a single responsibility
   - Minimize nesting and complexity
   - Remove unused code and avoid duplication

## When Working on This Project

- Always ask for clarification if requirements are ambiguous
- Suggest only the most essential improvements when reviewing code
- Prioritize maintainability over premature optimization
- Test thoroughly before considering a feature complete

## Reference

Refer to this document at the beginning of each development session to maintain consistency and focus on the project's core principles.
